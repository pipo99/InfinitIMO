CreateDetail

This method is used to create a detailed log entry for a specific activity performed by a user. The method expects a valid LogEntryDetail object containing the necessary information such as IdLog, IdOrigem, Json, Endpoint, FullEndpoint, and ErrorMessage. The method requires the provided IdLog field to be non-null and non-empty. Before creating the log entry, the user's license is retrieved from the JWT token, and the connection string for the staging environment is set accordingly. If the required data is missing or invalid, the method returns a Bad Request response with an appropriate error message. If the log entry is successfully created, it returns a unique identifier for the new detailed log entry.

Authentication: This endpoint requires a valid Bearer token to be included in the Authorization header of the request. Make sure to obtain a valid token before making requests to this endpoint.

Close

This method is used to close a log entry based on the provided details and final result. The method expects a valid log object containing the necessary information such as IDLOG, Ok, and Error. The log object should have Ok set to 1 if the log operation was successful without any errors, otherwise set to 0. If Ok is set to 0, the Error property should contain the appropriate error code associated with the log operation. The method will update the log entry identified by the specified IDLOG with the provided final result and error details (if applicable). If the provided IDLOG is not found or if the log entry is already closed, the method will not perform any action, and the response will indicate success. Note: It is assumed that IDLOG uniquely identifies an open log entry that needs to be closed.

Authentication: This endpoint requires a valid Bearer token to be included in the Authorization header of the request. Make sure to obtain a valid token before making requests to this endpoint.

Request Body schema:

The log entry details to be closed.
